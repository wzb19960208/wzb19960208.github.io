SpringBoot项目启动方式

SpringBoot项目集成了Tomcat

IDEA启动

SpringBoot有个非常直接的入口类，在开发调试的时候非常方便，只要右键运行就好了

命令行启动

SpringBoot的Pom.xml文件里面添加了maven的SpringBoot插件，提供了mvn启动命令

Jar包

可以通过mvn package将项目打成jar包，然后通过java -jar启动

war包

war包不能通过直接修改pom文件然后直接打包，必须修改以下两个地方

  • 修改启动类

  • 添加tomcat依赖以及插件

SpringBoot项目属性配置

  • 配置文件在resource文件夹下
  • static 用来放css/js等东西的
  • template 用来放渲染的html模板的
  • 这俩文件夹可以删了，后端用不到
  • 一般只会使用application.properties/application.yml

使用配置文件的变量

使用单个变量

  • 在属性文件里面声明
  • 在类里面使用@Value(“${}”)注入

通过类来填充

  • 使用前缀注解，然后会把这个注解下的所有属性注入进去
  • 这个注解要添加依赖

多个配置文件切换

  • 有时候需要切换线上和测试环境的配置文件
  • 需要一个总的配置文件properties
  • 其他的配置文件命名需要规范

LogBack配置

  • SpringBoot集成的日志框架

  • logback-spring.xml

  • 每天打印一个日志
  • 日志输出为Error/Info日志
  • 日志保存时间为7天，自己定的，避免了太多日志
  • 开发调试的时候，直接注释调配置文件就好了，打印到控制台上面看
  • 在服务器跑的时候，才需要日志文件
  • 用以下模板就好了，不需要手写

子查询

  • 因为最后都是通过一个类，来存储数据的
  • 只要写好SQL语句，返回需要的列，和类成员对应即可，不管SQL多复杂

配置数据库

配置MyBatis

建立数据库表映射类

  • 也就是配置mybtis的时候aliases-package下的类
  • 因为dao层通过SQL语句写的，不需要再在这里使用注解指定具体的表了
  • 主要就是用来存对应SQL语句的结果数据
  • 提供 构造函数/getter/setter 就ok了

Dao层

  • 使用dao访问数据库
  • 提供最简单的增删改查操作，使用和存储结果的对象是domain包下面的类
  • dao层是接口，具体实现由MyBatis实现了
  • 在接口上面使用MyBatis注解
  • 其实最返回的结果用哪里的类都行，只是建议写在domain下而已

Service层

  • 只是为了不要直接使用dao层
  • 并且service层里可能会有更复杂的逻辑操作，比如多次访问数据库
  • 在service层直接实现也可以，通过Ipml实现也可以，具体的实现要注入到，也就是加上@Service注解

使用AOP

实例 打印请求的IP和URL

全局异常处理

  • 主要是解决了在controller里面需要写很多逻辑代码，来选择错误情况的返回
  • service里面如果出错，那ontroller怎么返回呢？
  • service里面是没有办法直接返回Result的，必须在controoler里面拿到返回值，再返回，这就很多代码了

先继承一个RuntimeException

  • 关于Java的异常 https://www.cnblogs.com/sargeles/p/6691383.html
  • RuntimeException才是我们需要抛出的

创建异常处理类

  • @ ControllerAdvice注解 可以在Controller之前返回，这样就能返回Result了
  • @ ResponseBody 注解
  • @ ExceptionHandler(value = Exception.class)

RESTFul接口

返回的结果要统一

  • 返回一个统一的JSON串
  • 使用Result类转JSON，Spring里面使用@ResponseBody注解，返回类会转为JSON的
  • 提供成功/失败的方法

各种类

  • VO 和前端交互的类。因为从数据库取出来的实体，前端不是所有数据都需要的，这时候可以新建一个数据库实体的子集，再返给前端
  • DAO 对数据库进行增删改查的类
  • PO 持久层的类 对于数据库实体，没有任何操作，使用setter/getter
  • DTO 数据传输类 不同服务或不同分层之间的数据传输和转换

单元测试

  • dao和service层，是可以在本地测试的
  • 单元测试很重要！相当于验证每个齿轮的正确性！
  • 验证能否正确增删改查/能否完成具体的service等等，都是基础操作！
  • 具体的controller，可以使用postman测
  • 右键要测方法，然后调到test类下即可
  • test类是在test包下的，test包和main同级
  • 添加注解，必须！@SpringTest/@Runwith(SpringRunner.class)
  • @ Test方法可以单独运行,不需要在类里面通过main方法跑什么的
